Kumite Técnico #1

Paradigmas: Estructurado vs. Objetos

"¿Qué enfoque garantiza una evolución segura para un sistema que cambia mensualmente?"

GRUPO A: Estructurado

  • • Eficiencia y bajo consumo.
  • • Flujo lineal previsible.
  • • Cercanía al hardware.

GRUPO B: Objetos

  • • Encapsulamiento del cambio.
  • • Extensibilidad vía polimorfismo.
  • • Ocultamiento de complejidad.
Kumite Técnico #2

Arquitectura: Estándar vs. Limpia

"¿Priorizar el lanzamiento inmediato (MVP) o la robustez de mantenimiento a largo plazo?"

GRUPO A: Estándar

  • • Menor costo inicial.
  • • Estructura directa.
  • • Curva de aprendizaje mínima.

GRUPO B: Limpia

  • • Independencia tecnológica.
  • • Fácil de testear aisladamente.
  • • Negocio desacoplado.
Kumite Técnico #3

Estructuras: Array vs. Lista

"Sistema de turnos médicos con límite fijo de pacientes: ¿Memoria estática o dinámica?"

GRUPO A: Array (Estático)

  • • Acceso O(1) instantáneo.
  • • Menor consumo de RAM.
  • • Uso eficiente de caché.

GRUPO B: Lista (Dinámica)

  • • Flexibilidad de tamaño.
  • • Inserción veloz.
  • • Memoria no contigua.
